# Ideation And Scoring

Use this reference for idea generation, idea improvement, and rubric-based comparison.

## Idea Generation Pattern

Generate ideas as concrete moments, not abstract products. Each idea should include:

- User and situation.
- Trigger moment.
- Current pain or missed opportunity.
- AI role.
- Interaction in 1-3 steps.
- Demo path.
- Required data/API.
- Why it fits the rubric.
- Main risk and fallback.

Cover multiple tones:

- Practical utility.
- Emotional support or confidence.
- Social/collaborative.
- Playful or shareable.
- Technically impressive.
- Low-risk/high-polish.

## Improvement Checklist

When improving an existing idea, strengthen in this order:

1. User moment: make the "when" unmistakable.
2. Payoff: show the before/after in one screen or one flow.
3. AI value: make AI necessary, not decorative.
4. Demo clarity: make a stranger understand it in under 30 seconds.
5. Scope: remove features that do not improve the live demo.
6. Differentiation: name the product's sharpest contrast with normal tools.

## Default Scoring

Use official rubrics first. When unavailable, score 1-5:

| Dimension | Meaning |
|---|---|
| Insight | Real, specific user problem or moment |
| Feasibility | Can ship and rehearse in time |
| AI/Technical Depth | AI or engineering is meaningful and visible |
| Experience Completeness | End-to-end flow feels finished |
| User Value | Clear improvement for the user |
| Innovation | Fresh angle, not a generic wrapper |
| Demo Clarity | Easy to explain and show live |

Weighted score formula:

```text
sum(score * weight) / sum(weight)
```

If no weights are provided, use equal weights.

## Recommendation Rule

Do not automatically choose the highest raw technical ambition. Recommend the idea with the best combined chance of:

- Scoring well.
- Being built in time.
- Being memorable in a live demo.
- Matching the team's actual skills.
