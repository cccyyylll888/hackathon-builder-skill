# Demo, Poster, And Pitch

Use this reference to prepare the public-facing hackathon package.

## Demo Design

Build a demo route as a story with buttons:

1. Start with the user's painful moment.
2. Trigger the product's core input.
3. Show AI or system processing briefly.
4. Reveal a useful output.
5. Let the user take one action or see one consequence.

Prepare:

- Primary live path.
- Offline/mock fallback.
- Reset route.
- Sample data and screenshots.
- One sentence for every screen.

## Poster Structure

Poster copy should be scannable from 2 meters:

- Product name.
- One-line value proposition.
- Target user and moment.
- Three-step visual flow.
- Key AI/technical capability.
- Demo QR/link.
- Team name.

Avoid long paragraphs. Use concrete verbs and visible outcomes.

## Roadshow Script

Recommended structure:

- 0:00-0:20 hook: user pain and product promise.
- 0:20-1:30 live demo.
- 1:30-2:10 architecture and AI/technical depth.
- 2:10-2:40 impact, rubric fit, and differentiator.
- 2:40-3:00 limitations and future work.

## Judge Q&A Bank

Prepare answers for:

- Why is this not just a chatbot?
- What did you build during the hackathon?
- What happens when the AI/API fails?
- What data do you need and how do you handle privacy?
- Who would use this repeatedly?
- How would this scale after the hackathon?
- What would you build next with one more week?
